03

Cookie 使用说明

本插件在 manifest.json 中声明了 cookies 权限。以下是我们读取 Cookie 的具体场景：

🍪 为何需要读取 Cookie

  • 115 网盘 Cookie：调用 115 网盘离线下载 API（uplb.115.com）时， 需携带登录 Cookie 进行身份验证，这是 115 网盘 API 的强制要求。
  • 夸克网盘 Cookie：调用夸克网盘 Aria2 导出 API 时，需携带对应的登录 Cookie。

🔒 Cookie 的处理方式

  • Cookie 从浏览器本地读取，仅在本地内存中处理，不写入任何数据库。
  • Cookie 值仅被转发至对应平台的官方 API 地址，绝不发送至插件开发者的服务器。
  • 插件不会修改、删除或创建任何 Cookie。
  • 读取范围仅限于 *.115.com、*.quark.cn、*.terabox.app 等在 host_permissions 中明确声明的域。
⚠️
重要说明 本插件不使用广告 Cookie、跨站追踪 Cookie，也不会在您访问的网站上注入任何追踪脚本。 cookies 权限的唯一使用目的是获取已登录账户的身份凭证以完成离线下载。

数据使用目的

我们收集上述数据的目的如下：

核心功能（必要数据）

  • 将磁力链接、SHA1 哈希链接提交至 115 网盘进行离线下载
  • 通过 Aria2 接口导出 115/夸克/Terabox 网盘下载任务
  • 读取网盘目录结构以选择下载路径

增值服务（可选数据，需您授权）

  • 会员资格验证：通过账号 UID 验证您是否为插件付费会员，以开放 Aria2 等高级功能
  • 个性化服务：在插件界面展示您的用户名和头像
  • 资源索引：将您成功离线的磁力链接（仅哈希摘要，不含文件内容）同步至公共索引，供其他用户秒传使用
  • 服务改进：通过汇总统计分析（匿名）改善产品体验

06

数据存储与安全

本地存储

插件配置（Aria2 地址、播放器设置、通知配置等）存储于浏览器的 chrome.storage.local，数据只存在于您的本地设备， 其他网站和扩展无法访问。

服务器存储

  • 传输采用 HTTPS/TLS 1.2+ 加密
  • 账号标识数据在静态存储时采用 AES 加密
  • 服务器访问日志保留 90 天后自动清除
  • 定期进行安全审计和渗透测试

安全措施

  • 插件遵循 Chrome 扩展最小权限原则，仅申请功能必需的权限
  • Content Security Policy（CSP）配置为 script-src 'self'，防止 XSS 攻击
  • 不在代码中硬编码任何密钥或凭证
⚠️
互联网上没有绝对安全的数据传输方式。我们会采取行业标准的保护措施， 但无法对数据安全做出百分之百的保证。如果您发现安全漏洞， 请通过 GitHub Issues 向我们负责任地披露。
